# NASDAQ 综合指数历史数据分析 — 开发文档

## 项目概述

基于 NASDAQ 综合指数 1971-2026 年历史数据，构建交互式深度分析仪表盘。在已有年度收益率分析（`analysis.py` + `visualization.py`）的基础上，新增风险指标、技术分析、季节性分析、牛熊周期四大模块，最终通过 Plotly 生成一个自包含的 HTML 交互式仪表盘。

## 数据源

**文件：** `E:\cc\maxs 01\nasdaq_complete_1971_2026.csv`

| 字段 | 类型 | 说明 |
|------|------|------|
| `date` | string | 交易日期，格式 `YYYY-MM-DD` |
| `open` | float | 开盘价 |
| `high` | float | 最高价 |
| `low` | float | 最低价 |
| `close` | float | 收盘价 |
| `volume` | int | 成交量 |
| `amount` | int | 成交额 |

- **总行数：** 13,941 行（1971-02-05 至 2026-05-22）
- **历史最低：** 54.87（1974-10-03）
- **历史最高：** 26,635.22（2026-05-14）

### 数据质量分界

| 时间段 | 数据来源 | 质量说明 |
|--------|----------|----------|
| 1971-02-05 ~ 2003-12-31 | FRED | 仅收盘价有效，OHLC 四价相同，成交量=0 |
| 2004-01-01 ~ 2026-05-22 | 新浪财经 | 完整 OHLCV 数据（4 行零成交量异常值需前向填充） |

**开发约束：** 凡涉及 OHLCV 的分析（技术指标、成交量分析）必须限定在 2004 年之后。

### 模块一：`risk_metrics.py` — 风险指标分析

**职责：** 计算并输出关键风险度量指标。

**数据依赖：** 仅需 `close` 列，全量 1971-2026 有效。

#### 函数列表

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `compute_max_drawdown(prices)` | pd.Series | dict | 计算最大回撤及 peak/trough 日期 |
| `compute_annualized_volatility(prices, window=252)` | pd.Series | float | 年化波动率（日对数收益率 × √252） |
| `compute_sharpe_ratio(prices, risk_free_rate=0.02)` | pd.Series | float | 夏普比率 |
| `compute_consecutive_years(yearly_returns)` | pd.DataFrame | dict | 最长连涨/连跌年数及区间 |
| `compute_bear_markets(prices, threshold=-0.20)` | pd.Series | DataFrame | 所有回撤超 20% 的熊市区间 |
| `compute_drawdown_series(prices)` | pd.Series | pd.Series | 完整回撤时间序列（供仪表盘绘图） |

#### 最大回撤算法

```
running_max = prices.expanding().max()
drawdown = (prices - running_max) / running_max
max_dd = drawdown.min()  # 返回负数
```

#### 熊市识别逻辑

1. 计算每日回撤序列
2. 找到回撤 < -20% 的起始点
3. 跟踪至回撤恢复到 0（价格回到前高）或创出新高
4. 记录每个熊市区间的起止日期、最大回撤、持续天数

#### 金叉/死叉检测

```python
# 金叉：SMA50 从下方穿越 SMA200
# 死叉：SMA50 从上方穿越 SMA200
crosses = []
for i in range(1, len(ohlcv)):
    if prev_diff < 0 and curr_diff >= 0:
        crosses.append({'date': ..., 'type': 'golden'})
    elif prev_diff > 0 and curr_diff <= 0:
```

```python
monthly_close = df.set_index('date')['close'].resample('ME').last()
```

### 模块四：`cycle_analysis.py` — 牛熊周期分析

**职责：** 识别和分析市场周期。

**数据依赖：** 仅需 `close` 列，全量 1971-2026 有效。

#### 函数列表

| 函数 | 输出 | 说明 |
|------|------|------|
| `identify_bull_bear_markets(prices, threshold=-0.20)` | DataFrame | 牛熊区间列表 |
| `compute_cycle_statistics(cycles)` | dict | 牛熊平均持续时间、收益率等统计 |
| `compute_recovery_analysis(cycles)` | dict | 熊市复苏时间分析 |
| `identify_current_regime(prices, cycles)` | dict | 当前所处市场状态 |

#### 牛熊识别算法

```
状态机：
  BULL → 当回撤达到 -20% 时 → BEAR
  BEAR  → 当价格回到前高时 → BULL

记录每个状态区间的：
  - start_date, end_date
  - duration_days
  - start_price, end_price
  - return_pct
  - period_type: 'bull' / 'bear'
```

#### 1. 2000-2002 互联网泡沫破裂（-79.7%，史上最大）

**周期：** 2000-03-10 → 2002-10-09（922 个交易日）
**高点：** 5,048.62 → **低点：** 1,004.95
**恢复：** 2015-04-23（历时 15 年）

**历史背景：**
- 1990 年代末，互联网公司疯狂上市，任何带 ".com" 的公司都能获得天价估值
- 纳斯达克从 1995 年的 1,000 点一路飙到 2000 年 3 月的 5,048 点
- 典型荒谬案例：Pets.com、Webvan 等无盈利公司估值数十亿美元
- 1999 年纳斯达克单年暴涨 85.6%，市场极度狂热

**基本面崩塌：**
- **盈利证伪：** 绝大多数互联网公司没有盈利模式，烧钱获取用户但无法变现
- **估值泡沫：** 市盈率普遍在 100x-500x，甚至无法用市盈率估值（因为利润为负）
- **资金链断裂：** 风险投资枯竭，IPO 市场关闭，新公司无法融资
- **企业破产潮：** Pets.com（2000-11）、Webvan（2001-07）、WorldCom（2002-07）等巨头倒塌
- **会计丑闻：** 安然（2001-12）、世通（2002-06）财务造假曝光，市场信任崩塌

**政策/宏观因素：**
- 美联储 1999-2000 年加息（从 4.75% 到 6.5%）收紧流动性
- 9·11 事件（2001-09）进一步打击市场信心
- 企业资本支出大幅缩减，科技行业进入寒冬

**关键事件时间线：**

|------|------|

| 2001-12-02 | 安然申请破产，市值蒸发 740 亿美元 |
| 2002-06-25 | 世通公司财务造假曝光 |

| 2021-11-03 | 美联储宣布开始缩减购债（Taper） |

| 2022-03-16 | 美联储开启加息周期（首次加息 25bp） |
| 2022-06-15 | 美联储单次加息 75bp，为 1994 年以来最大 |
| 2022-09-13 | CPI 超预期，纳斯达克单日暴跌 5.2% |
| 2022-11-08 | FTX 交易所破产 |
| 2022-12-28 | 纳斯达克触及年度低点 10466 |

**教训：** 估值杀（利率上升压低估值）与盈利杀不同，基本面未崩但流动性急剧收紧。

#### 5. 1987 黑色星期一（-35.9%）

**周期：** 1987-08-26 → 1987-10-28（63 个交易日）
**高点：** 455.26 → **低点：** 291.79
**恢复：** 1989-08-03（历时 708 天）

**历史背景：**
- 1987 年 10 月 19 日，道琼斯单日暴跌 22.6%（史上最大单日跌幅）
- 纳斯达克同日暴跌约 11%

**基本面/技术面因素：**
- **组合保险（Portfolio Insurance）：** 机构大量使用自动止损策略，下跌触发连锁抛售
- **程序化交易：** 计算机自动卖出加剧了跌势
- **估值过高：** 1987 年前 8 个月市场涨了 40%，获利盘丰厚
- **利率上升：** 美联储 1987 年收紧货币政策
- **贸易赤字：** 美国贸易逆差扩大，美元贬值担忧

**关键事件时间线：**
| 日期 | 事件 |
|------|------|
| 1987-08-26 | 纳斯达克触及 455 高点 |
| 1987-10-14 | 道琼斯单日跌 3.8%（程序化卖单启动） |
| 1987-10-16 | 道琼斯再跌 4.6%（周末恐慌发酵） |
| 1987-10-19 | 黑色星期一，道琼斯单日暴跌 22.6% |
| 1987-10-28 | 纳斯达克触及低点 291 |

**教训：** 技术面/流动性驱动的崩盘通常恢复较快，因经济基本面未受损。

#### 6. 2020 新冠疫情闪崩（-30.1%，最快暴跌）

**周期：** 2020-02-19 → 2020-03-23（23 个交易日）
**高点：** 9,817.18 → **低点：** 6,860.67
**恢复：** 2020-06-08（仅 110 天）

**历史背景：**
- 2020 年 2-3 月，新冠疫情全球蔓延
- 3 月 9-18 日，美股 4 次熔断（史上首次）

**基本面冲击：**
- **经济停摆：** 全球封锁，GDP 断崖式下跌
- **不确定性爆炸：** 病毒传播、死亡人数、经济影响均未知
- **流动性危机：** 所有资产同时下跌，连黄金和国债都被抛售（美元荒）
- **企业前景恶化：** 航空、酒店、零售等行业收入归零

**为什么恢复最快：**
- 美联储史无前例的反应速度：3 月 15 日降息至 0 + 无限 QE
- 财政刺激：CARES Act 2.2 万亿美元
- 科技股受益于居家办公（Zoom、Netflix、Amazon 等逆势上涨）

**关键事件时间线：**
| 日期 | 事件 |
|------|------|
| 2020-02-20 | WHO 命名 COVID-19 |
| 2020-03-09 | 美股第一次熔断 |
| 2020-03-12 | 美股第二次熔断（黑色星期四） |
| 2020-03-16 | 美股第三次熔断（一周内第三次） |
| 2020-03-18 | 美股第四次熔断（史上首次） |
| 2020-03-23 | 纳斯达克触及低点 6860 |
| 2020-06-08 | 纳斯达克收复失地（仅 110 天） |

**教训：** 政策应对速度决定恢复速度，美联储史无前例的反应使恢复仅用 5 个月。

### 规律总结

| 模式 | 代表事件 | 平均恢复时间 | 核心驱动 |
|------|----------|--------------|----------|
| **泡沫破裂** | 2000-2002 | 10-15年 | 估值回归基本面 |
| **金融危机** | 2008 | 5-8年 | 信贷/银行系统崩溃 |
| **外部冲击** | 1973-74, 2020, 2025 | 1-6年 | 石油/疫情/关税 |
| **政策收紧** | 2022 | 1-2年 | 美联储加息 |
| **技术面踩踏** | 1987 | 2年 | 程序化交易 |

**关键观察：**
1. **跌幅越大，恢复越慢** — -80% 需要 15 年，-30% 只需数月
2. **基本面驱动的下跌比情绪驱动的更深** — 泡沫破裂 > 金融危机 > 外部冲击
3. **政策应对速度决定恢复速度** — 2020 年（最快应对）恢复最快
4. **估值越高，下跌越猛** — 2000 年（PE 100x+）和 2021 年（PE 40x+）都是高估值起跌
